dispatch(targetID, action) — the new public method. It wraps the action in a JSON envelope with targetID, senderID (always this client's Config.appID), and the action string, then hands it to Socket.send(). Examples:
js// target a specific client
GarageApp.Events.dispatch(2, "GarageApp.Player.jumpTo(30.0)");

// broadcast to every connected client
GarageApp.Events.dispatch(-1, "GarageApp.Player.pause()");
doEvent() is now envelope-aware — when a message arrives over the socket it first tries JSON.parse. If it finds the expected { targetID, senderID, action } shape it routes to _handleEnvelope(); if it's plain text (legacy commands, local timed events) it falls through to _execute() as before — so nothing existing breaks.
_handleEnvelope() — the private gatekeeper. It applies two checks before executing: it drops any message whose senderID matches this client (server echo guard, since most WS servers broadcast back to the sender), and it only executes when targetID matches Config.appID or is -1. Everything else is silently ignored with a console note.
The envelope schema is documented at the top of the file so your server-side relay code has a clear contract to work against — the server itself just needs to broadcast whatever JSON string it receives to all connected clients; the filtering happens entirely on the client side.
